Installation et Choix Techniques

1. Concepts Théoriques

Définition de la virtualisation : C'est une technique permettant de faire fonctionner plusieurs systèmes d'exploitation (machines virtuelles) isolés sur une seule machine physique, en partageant les ressources matérielles.

Différences entre les outils :

  • Émulateur : Logiciel qui reproduit le comportement d'un matériel (console de jeu, ancien CPU). C'est lent car chaque instruction est traduite pour fonctionner sur notre machine.
  • Simulateur : Imite le comportement global d'un système (ex: simulateur de vol) sans forcément reproduire son fonctionnement interne exact.
  • Outil de Virtualisation : Isole un système d'exploitation sur le matériel réel. C'est le plus performant (ex: VMware, VirtualBox).

2. Environnement de Virtualisation

Type de virtualisation : Les logiciels VirtualBox et VMware Workstation sont des hyperviseurs de Type 2 (Hébergé). Ils s'installent comme une application sur un système d'exploitation (comme Windows), contrairement au Type 1 (comme Stratus ou ESXi) qui s'installe directement sur le matériel (Bare Metal).

Sauvegarde de la VM (Forme et Lieu) : La machine virtuelle créée n'est pas "dans" le logiciel, c'est un ensemble de fichiers stockés sur le disque dur de l'hôte.
Elle se trouve généralement dans Documents/Virtual Machines.
Elle est constituée principalement d'un fichier de configuration (.vmx ou .vbox) et d'un disque dur virtuel (.vmdk ou .vdi). C'est le .vmdk qui est utile pour lancer la machine virtuelle sur un autre logiciel que celui dans lequel on l'a configuré.

Justification détaillée des choix de configuration

Système d'exploitation : Ubuntu 64-bit

Nous avons opté pour la version LTS (Long Term Support) d'Ubuntu.
Justification : Ce choix garantit une stabilité maximale pour le développement. L'architecture 64 bits est indispensable pour faire tourner des outils modernes comme WebStorm qui ne sont plus maintenus en 32 bits. De plus, la gestion des paquets via apt facilite l'installation du serveur Apache.

Mémoire Vive (RAM) : 4 Go

Allocation de 4096 Mo (4 Go) sur les 16 ou 32 Go de la machine hôte.
Justification : C'est un point d'équilibre critique. Il faut suffisamment de RAM pour que l'interface graphique d'Ubuntu et l'IDE (IntelliJ/WebStorm) soient fluides, sans pour autant asphyxier le système hôte (Windows) qui doit continuer à fonctionner en parallèle.

Stockage : 20 Go (Split)

Création d'un disque virtuel de 20 Go découpé en plusieurs fichiers (Split).
Justification : 20 Go suffisent pour l'OS (~10Go) et les projets. L'option "Split virtual disk into multiple files" est cruciale pour la portabilité : elle permet de copier la VM sur une clé USB formatée en FAT32 (qui refuse les fichiers uniques de plus de 4 Go).

Réseau : NAT

Configuration de la carte réseau en mode NAT (Network Address Translation).
Justification : Ce mode permet à la VM d'accéder à Internet en utilisant l'adresse IP de la machine hôte. C'est le choix le plus simple pour télécharger des paquets (mises à jour, Apache) tout en protégeant la VM des connexions entrantes non sollicitées.

Phase 2 : Migration et Portabilité (Domicile)

Une fois l'installation terminée à l'université, je n'ai pas eu besoin de refaire toute la procédure chez moi. J'ai simplement récupéré le dossier de la machine virtuelle (contenant le fichier disque .vmdk).

Sur mon poste personnel avec VirtualBox :
J'ai créé une nouvelle machine virtuelle, mais à l'étape du disque dur, j'ai choisi l'option "Utiliser un fichier de disque dur virtuel existant". VirtualBox étant compatible avec le format VMware, j'ai pu reprendre mon travail exactement là où je l'avais laissé.
